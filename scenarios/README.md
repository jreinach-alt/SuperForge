# scenarios/ — the genre-rail catalog

This catalog is simultaneously the kit's **spec** (what it can build), the
**routing table** (where an ask goes), and the **acceptance bar** (the named
test suite IS each rail's done-condition — every rail is emulator-verified in
CI, every run).

**How to use it (the action loop, per `AGENTS.md`):**

1. **Clarify** a broad/ambiguous ask until it maps to a rail below.
2. **Route** to the nearest rail's template; pull capability bricks from
   other rails as needed (they compose — the flagship is proof).
3. **Adapt** with the macro library (`lib/macros/`) — never freeform-generate.
   To re-theme a whole rail into a new game, copy it and follow the concrete
   change-list in [`docs/guides/adapting_a_rail.md`](../docs/guides/adapting_a_rail.md).
4. **Verify** the rail's done-condition on the emulator before calling it done.

**Off-catalog asks:** if no rail or composition below fits, say so honestly —
name the closest rail and the unproven gap. See "Genre routing map" at the
bottom for where common asks land. The knowledge-realm catalog (hardware Q&A,
troubleshooting, expert interrogation) is in [`knowledge.md`](knowledge.md).

## The rails

Thirty-six templates (`templates/<name>/`): complete-game rails grown in
strict capability order — every later rail composes the earlier ones — plus
effect / primitive rails (the split-screen family, the Mode 7↔Mode 1
transition, the barrel chamber, the BG-mode showcase) and two kept-in-tree
cold-start trials. Three minimal teaching examples (`examples/`) sit below
them.

### Foundations (examples/)

| Rail | Demonstrates | Done-condition |
|---|---|---|
| `hello_world` | cold start → defined PPU state → text on screen. The smallest complete ROM. | boots; renders the text (`tests/test_examples.py`) |

| `move_sprite` | one OAM sprite moved by the d-pad | OAM X/Y track injected input, all 4 directions (`tests/test_examples.py`) |

**Composition ≠ rail.** A "composition" row means every brick is proven but
the specific assembly is not — say that to the user, build on the bricks, and
verify the composed done-condition on the emulator like any rail. The catalog
grows by promoting compositions that worked into rails.

**Post-v1.0 (explicitly deferred, no rail):** slopes, ladders, moving
platforms, combined-axis (8-way) level scrolling, >32×32 metasprites.
