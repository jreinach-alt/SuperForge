# SuperForge Kit — Roadmap

> **This is the roadmap for the SuperForge kit** — the clean-room SNES homebrew kit
> (materialized via `tools/dryrun_split.sh`), destined to ship as a **standalone public repo**.
> It tracks **only kit concerns**: the proven rails, the rail-promotion queue, and release
> readiness. It is the single roadmap for kit work; nothing else governs the kit's direction.

**Cross-refs (do not duplicate their state here):**
- **What the kit can build + each rail's done-condition** → [`../scenarios/README.md`](../scenarios/README.md)
  (the genre-rail catalog — the kit's spec, routing table, and acceptance bar in one).
- **How to operate the kit** → [`../AGENTS.md`](../AGENTS.md) (two realms, routing, rigor, the kit map).
- **Per-rail guides** → [`guides/`](guides/).

`hello_world` · `buttons` · `move_sprite` (examples) · `sprite_game` · `scroller` · `camera_follow` ·

`brawler` · `platformer` (flagship) · `racer` · `breaker` · `rpg` · **Mode 7 streaming**
(`mode7_explore`) · **Mode 1 streaming** (`platformer_stream`) · **Mode 7 free-flight**
(`mode7_flight`) · **Mode 7 rotating-floor dungeon** (`m7_dungeon`) · **Mode 7 overhead shooter**
(`m7_oshoot`).

The kit grows by **promoting compositions that worked into rails**. A promotion is done when the
genre's row in the catalog flips to **proven rail** with a named CI test suite as its done-condition.

## Release readiness (standalone public release)

The kit ships as a standalone open-source release once it is self-contained and complete. Criteria:

1. **Template set complete** — every intended genre has a proven rail (no "almost-covered" gaps the
   catalog has to apologize for).
2. **Self-contained** — no doc/guide/scenario/example reaches outside the kit tree; the full
   materialized kit builds + passes its entire suite + all gates (`width-check` / `zp-check` /
   `cleanroom_check`) with zero external dependency. (Every cold-start trial already proves the
   materialized kit builds working rails in isolation.)
3. **Public-release pass** — LICENSE (incl. CC0 art credits), CI running the gates + tests, a
   public-facing README / contribution guide, a tagged release.

*(The kit-line orchestration that gets us there — the template-harvest triage and the reach-back
audit — is tracked in the project's branch/topology notes, not in this product roadmap.)*
