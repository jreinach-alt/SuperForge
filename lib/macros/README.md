# lib/macros/ — the ca65 macro library (THE KEYSTONE)

> The asm-idiomatic API surface — **the single front door** of this repo. **Everything in the Action realm routes through it.** Proven on the A0→A1→A2 vertical slice (the build-and-test loop), two-audited.

**What it is:** ergonomic ca65 macros over the existing engine functions — one surface at three altitudes (a newcomer's agent *emits* them, a planner *writes* them, an expert *inspects* them). A macro sets up the API block and `JSR`s the engine-function wrapper; the calling convention underneath is the same thing at a lower altitude (resolves catalog conflict **C3**).

**The invariant principle (AX gauntlet = this library's correctness suite):** each hardware landmine is **baked into the macro** so the generated code is *right by construction* and a user can't accidentally get it wrong — e.g. the VRAM-upload macro inherently force-blanks + sets VMAIN; the controller-read macro waits on `$4212`; the Mode-7 matrix macros honor the write-twice latch. The agent's added value is surfacing the *why*, not remembering the gotcha.

**Game memory contract** (where YOUR state goes):

- **DP `$32-$5F`** — fast per-frame game variables (the API block owns `$60-$9F`; templates by convention start at `$32`). It is small: a player + two actors fills it.
- **WRAM `$1800-$1DFF`** — the game-array region: pools, parallel arrays, low-frequency scalars (`lda abs` reachable). Engine-free by construction; see `sf_pool.inc`'s header for why `$2000+` is NOT usable from bank $00.
- Macro scratch already claimed: `$0530-$0535` (sf_text), `$0536-$0539` (sf_meta), `$053A-$053D` (sf_physics), `$053E-$053F` (sf_math random state — persists between `sf_rnd` calls).
- Audio ROMs: TAD claims DP `$FE-$FF` and `$1DE0-$1DFF` (the game-array region's tail — don't put pools there; the kit's `lorom_tad.cfg` owns this placement).

**Usage notes:**

- **Macros expand inline — short branches can't jump over big ones.** A `beq/bne/bcc/bcs` over a macro call whose expansion is >127 bytes (e.g. the print macros, `sf_camera_follow`) fails with `Range error (N not in [-128..127])` reported at the expansion site. Use the trampoline idiom: invert the branch to a near target, `jmp` past the macro body (`beq @do` / `jmp @skip` / `@do: <macro>` / `@skip:`).
- **Macros with `.local` labels end cheap-local `@` scope.** `.local` symbols expand to real (uniquely renamed) labels, and ca65 scopes `@labels` between real labels — so an `@loop` defined before a call to `sf_solid_box`/`sf_physics_step`/`sf_jump` can't be referenced after it (`Symbol '@...' is undefined`). Use a named label for any branch that spans such a macro call.
